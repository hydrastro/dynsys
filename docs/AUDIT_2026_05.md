# Codebase audit (May 2026) — findings & fixes

A general health pass before the Lizard/Phase-E work. Summary: the codebase
is in good shape; one real bug (IFS parameters) is fixed, plus three small
robustness/cleanup items. No memory or undefined-behavior issues found.

## Fixed: IFS couldn't have parameters (the reported bug)

The IFS compile branch hardcoded `app.params.clear()` and parsed map
coefficients as plain numbers (std::stod), so an IFS literally could not be
parametrized. Fixed by:
- parsing `param name = value [min,max]` lines in IFS mode (kept, not
  cleared);
- treating each `ifs_map` field as an EXPRESSION (parsed via the real IR),
  so coefficients can reference parameters, e.g.
  `ifs_map = s*cos(theta), 0 - s*sin(theta), s*sin(theta), s*cos(theta), 0, 0, 0.5`;
- evaluating those expressions against the current parameter values at
  render time (evaluate_ifs_maps), so dragging a parameter changes the
  attractor live.
New preset "Spiral IFS (parametrized)" demonstrates it (drag theta and s).
Verified: test-ifsparam (parameter changes the attractor, stays bounded,
sane dimension) and an IR parse check of the coefficient expressions.

## Audit findings (all addressed)

1. **Map-vs-ODE branches and IFS.** Checked every `mode == Map ? : (ODE)`
   branch. The per-frame integrator already skips IFS; the only issue was a
   debug printf showing "ode" for an IFS — now uses mode_name() ("ifs").
2. **Parse crashes.** All std::stod/stoi calls are inside try/catch
   (including the new IFS param-range parse). No unguarded numeric parses.
3. **Out-of-bounds indexing.** The `params[0]` / `state_names[0]` /
   `lcc_pp.front()` accesses are each guarded by an earlier emptiness check
   or a `!param` fallback. No OOB paths found.
4. **IFS lifetime.** The IFS expression arena is now freed on shutdown and
   when switching away from an IFS system (was a minor leak / potential
   stale pointer); IFS maps/expressions are cleared when a non-IFS system
   compiles.

## Clean bills of health

- **Sanitizers**: analysis core + all analysis-linked tests
  (box-counting, IFS, limit-cycle, LC sweep) run clean under
  AddressSanitizer + UBSan — no leaks, no UB.
- **Warnings**: all 4 translation units compile at -O2 -Wall -Wextra with
  ZERO warnings.
- **Tests**: 24 suites/checks pass (added test-ifsparam this pass).
- **Presets**: 44 total; the parametrized-IFS coefficient expressions parse
  and lower through the real expression IR.
